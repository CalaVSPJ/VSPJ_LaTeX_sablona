\thispagestyle{fancy}
\addcontentsline{toc}{chapter}{Závěr}
Verzovací software, ať už git, o~kterém pojednává práce, nebo jeho méně známe alternativy (Mercurial, Subversion) jsou rozhodně nástroje, které spadají do potenciálního repertoáru každého programátora. Jejich znalost je velkým přínosem pro samotného uživitale, ale také pro celý proces softwarového inženýrství. A~je v~podstatě nutností v~případě práce ve větším týmu.

Přidružená funkcionalita poskytovaná platformami jako GitHub aj. rozhodně důvody pro využívání VCS jen rozšiřuje, a tak se i samotný git stává vlastně téměř nezbytností.

Z praktické zkušenosti, demonstrované i v rámci práce, mohu říci, že git je nestárnoucí a~jednoduše uchopitelný nástroj, který má velké využití i~v~dnešní době, a~díky dostupné dokumentaci, je jeho používání jednodušší než kdykoliv dříve. Poskytuje navíc velmi unikátní funkcionalitu, kterou alternativní nástroje doposud nezahrnují.
